## 1. Project Overview

**The Future of AI Is Dead** is a *complete, self-contained digital product business* built as a Next.js 16 web application. It ships with:

- A 10-chapter ebook exposing AI hype and teaching a practical "Hybrid Intelligence" framework for CEOs and operators.
- An interactive in-browser flipbook reader (free preview for the first two chapters).
- A high-converting multi-stage sales funnel — from free lead magnet through four paid upsells.
- An exit-intent down-sale popup that saves abandoning visitors.
- A 50% affiliate program with real-time tracking and a self-serve dashboard.
- Master Resell Rights (MRR) license system for both the ebook and the complete software.

**Who is this for?**

| Use Case | What you do |

| End reader | Buy the $9.99 ebook, read it, use the framework |

### Stage 9 — Exit-Intent Down-Sale (Free Chapters / Discount)

**Component:** `ExitIntentDownsale` (`src/components/ExitIntentDownsale.tsx`)
**Active on:** `/checkout`, `/offer`, `/funnel`

The exit-intent popup fires when the user's mouse cursor exits through the **top** of the browser viewport — the universal signal for "I'm about to close this tab or navigate away." It renders a full-screen modal with a compelling down-sell offer calibrated to the page context:

| Page | Down-Sale Offer |
|---|---|
| `/checkout` | Free chapters access — capture the email before losing them entirely |
| `/offer` | Free chapters access — let them sample before losing the sale |
| `/funnel` | 20% discount ($7.99 instead of $9.99) — price objection crusher |

**Technical behaviour:**
- 3-second delay on page load prevents false triggers during scrolling
- Does not fire on touch/mobile devices (mouse events are not relevant)
- Once dismissed it does not re-fire in the same session
- Animated entry/exit via Framer Motion spring transition
- Fully accessible: `role="dialog"`, `aria-modal`, labelled heading, close button with `aria-label`

**Why it works:**

Most funnels simply let the customer leave. The exit-intent popup gives you one final chance to convert at a lower commitment level (free content) or a lower price point (discount). Even if the visitor doesn't buy today, capturing their email at this stage means you can follow up via automated email sequence and close the sale later.

## 5. Affiliate Program

**Route:** `/affiliate` (sign-up), `/affiliate/dashboard` (tracking)
**API:** `POST /api/affiliate/register`, `POST /api/affiliate/track`

### How it works

1. Visitor navigates to `/affiliate` and enters their email.
2. `POST /api/affiliate/register` generates a unique `AFF-XXXXXXXX` code and returns a tracking URL: `https://thefutureofaiisdead.com/?ref=AFF-XXXXXXXX`.
3. When someone visits the site via that link, a 30-day affiliate cookie is set.
4. When a purchase is made, the Stripe webhook reads the `affiliateCode` from session metadata and calculates the commission.
5. Affiliates log in to `/affiliate/dashboard` to see clicks, conversions, total earned, and pending payouts.

### Commission Structure

| Sales | Earned |
|---|---|
| 10 sales | $50 |
| 100 sales | $500 |
| 1,000 sales | $5,000 |

- Commission: **50%** ($5 per $9.99 sale)
- No caps, no approval delays
- 30-day cookie window
- Instant tracking

## 6. License Tiers & Resell Rights Framework

The system supports a four-level value ladder with two distinct resell license products:

```
LEVEL 1 — Reader / End Customer
  Purchase: $9.99 ebook
  Rights: Personal use only

LEVEL 2 — Ebook Reseller (Ebook MRR, $49)
  Purchase: $49 Ebook MRR license
  Rights: Sell the ebook, keep 100%. Sell MRR rights to others.
  Minimum price floor: $7.00/copy

LEVEL 3 — Software Reseller (Software MRR, $149)
  Purchase: $149 Software MRR license
  Rights: White-label + rebrand + deploy the entire funnel.
         Sell the software to clients or as SaaS.
         Sell the software MRR license to your own customers (one level down).
  Minimum resale price: $99/license
  Domain activations: Up to 3 domains per license

LEVEL 4 — Master Reseller (Both Licenses)
  Purchase: Ebook MRR ($49) + Software MRR ($149) = $198 combined
  Rights: Own the entire system — ebook + software.
         Use the funnel for multiple products (swap out the ebook content).
         Sell the ebook. Sell the software. Sell both MRR licenses.
         Run the funnel as a standalone business for ANY digital product.
```

### License management

Licenses are managed at `/dashboard/licenses`. Each license card shows:
- License ID (e.g. `LIC-EBOOK-001`)
- License key (e.g. `AI-EBOOK-MRR-XXXX-XXXX`) with one-click copy
- Active/inactive status
- Number of domain activations used (software license)
- Whether resale rights are included

## 10. API Endpoints

### `POST /api/signup`

Captures a lead (name + email) with source attribution.

**Request body:**
```json
{ "name": "Jane", "email": "jane@example.com", "source": "funnel-free" }
```

**Response:**
```json
{ "success": true, "message": "Welcome! Your free access is on its way." }
```

**Integration:** Replace the `console.log` in `src/app/api/signup/route.ts` with your email provider SDK call.

---

### `POST /api/checkout/create`

Creates a Stripe Checkout Session and returns the hosted checkout URL.

**Request body:**
```json
{ "email": "jane@example.com", "includeBump": true, "affiliateCode": "AFF-ABC123" }
```

**Response:**
```json
{ "url": "https://checkout.stripe.com/pay/cs_..." }
```

**Fallback:** If Stripe is not configured, returns a direct URL to `/upsell/starter` so the funnel still works without payment processing.

---

### `POST /api/checkout/webhook`

Handles Stripe webhook events. Validates signature with `STRIPE_WEBHOOK_SECRET`.

**Events handled:**
- `checkout.session.completed` — creates order, entitlement, and affiliate commission records (TODO: connect to DB)
- `customer.subscription.deleted` — cancels membership (TODO: connect to DB)

---

### `POST /api/affiliate/register`

Registers a new affiliate and returns their unique referral code and tracking link.

**Request body:**
```json
{ "email": "affiliate@example.com", "name": "John" }
```

**Response:**
```json
{
  "success": true,
  "referralCode": "AFF-3FA2C1",
  "affiliateLink": "https://thefutureofaiisdead.com/?ref=AFF-3FA2C1",
  "message": "Your affiliate account is ready! Start sharing your link."
}
```

---

### `POST /api/affiliate/track`

Logs an affiliate link click (for dashboard stats).

**Request body:**
```json
{ "referralCode": "AFF-3FA2C1", "sessionId": "sess_xyz" }
```

Exit-intent popup component. Detects cursor exiting the viewport top edge and displays a contextual down-sell offer.


**Behaviour:**
- 3-second activation delay (prevents false triggers on page load)
- Mouse-leave-top-of-viewport detection
- Once-per-session (dismissed state prevents re-trigger)
- Fully accessible modal (ARIA attributes, focus management)
- Does not fire on touch/mobile devices

## 17. Security & Compliance Notes

- **Stripe webhook signature verification** is enforced in `/api/checkout/webhook/route.ts`. Do not remove this check.
- **Email validation** is performed server-side in all API routes using a regex before any processing.
- **Affiliate codes** are generated using `crypto.randomBytes` (Node.js built-in) — not `Math.random()`.
- **No secrets are committed** — all sensitive values use environment variables.
- **HTTPS only in production** — Stripe requires HTTPS for all production webhook endpoints and redirect URLs.
- **MRR license terms** at `/license/ebook-mrr` and `/license/software-mrr` include liability limitations and governing law clauses. Consult a solicitor before reselling in your jurisdiction.
- The `ExitIntentDownsale` component does not collect or transmit any data. It is purely a UI interaction.
